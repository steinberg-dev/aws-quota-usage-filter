Implementation

API-level filter applied before GetMetricStatistics calls — skips quotas with no UsageMetric entirely

Key insight: The CloudWatch cost comes from GetMetricStatistics calls, not from ListServiceQuotas. Filtering at the list stage eliminates the downstream CW calls entirely — not just the metric count.

Per-service config format

As discussed in #213, a global flag is too blunt — some services may need full quota collection. The implementation uses per-service config:

config.yaml
collect: only_with_usage: # Global default — can be overridden per service default: false services: ec2: true # 1,743 → 31 metrics rds: true # 312 → 23 metrics lambda: true # 89 → 6 metrics s3: false # keep all S3 quotas # full service list + validation in implementation branch
