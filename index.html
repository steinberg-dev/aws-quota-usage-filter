Filter AWS Quotas to Usage-Only Metrics

Reduces scrape time and Prometheus metric count by collecting only quotas that have a CloudWatch usage metric defined.

Go 1.22 Prometheus AWS Service Quotas MIT License

Overview

Not all AWS quotas have a corresponding CloudWatch usage metric. Collecting unused quotas inflates scrape time and metric count with no observability benefit.

AWS EC2 alone has 1,742 service quotas, but only 31 have a UsageMetric defined. The other 1,711 generate CloudWatch API calls that return no usable data.

Without filter — EC21,742 quotas · 94s scrape
With filter — EC231 quotas · 3.1s scrape

Configuration

Per-service flag in the existing config file. Mix filtered and unfiltered services freely.
# aws-quota-exporter config with usage filter
quotas:
  ec2:
    collect_only_with_usage: true    # 1742 → 31 quotas
  lambda:
    collect_only_with_usage: true    # 312 → 18 quotas
  rds:
    collect_only_with_usage: true
  ecs:
    collect_only_with_usage: false   # keep all quotas for this service
  s3:
    collect_only_with_usage: true

scrape_interval: 5m
regions:
  - us-east-1
  - eu-west-1

Installation

# Install
go install github.com/steinberg-dev/aws-quota-usage-filter@latest

# Or build from source
git clone https://github.com/steinberg-dev/aws-quota-usage-filter
cd aws-quota-usage-filter
go build -o aws-quota-usage-filter .

# Run alongside existing exporter
./aws-quota-usage-filter --config=config.yaml --port=9200
